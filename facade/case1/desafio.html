Desafio — Case 1

Smell Code para refatorar

Contexto

Um sistema de e-commerce precisa processar um pedido. Para isso, o Main coordena diretamente quatro subsistemas: verifica o estoque, processa o pagamento, emite a nota fiscal e envia o e-mail de confirmação ao cliente.

O problema é que toda essa orquestração está exposta no cliente (Main). Qualquer outro ponto do sistema que precise fazer um pedido terá que duplicar essa sequência de chamadas — e qualquer mudança em um subsistema afeta todos os clientes.

Sobre o smell code

Subsistema Complexo Exposto Diretamente

O Main conhece e chama diretamente todos os subsistemas na ordem correta. Isso cria alto acoplamento: o cliente precisa saber como o pedido funciona, não apenas que quer fazer um pedido.

Se a ordem das etapas mudar, ou um novo subsistema for adicionado, todos os clientes precisam ser atualizados.

Código atual

smell code 
public class ServicoEstoque {
    public boolean verificarDisponibilidade(String produto, int quantidade) {
        System.out.println("Estoque: verificando " + quantidade + "x " + produto);
        return true;
    }

    public void reservar(String produto, int quantidade) {
        System.out.println("Estoque: reservando " + quantidade + "x " + produto);
    }
}

public class ServicoPagamento {
    public boolean processar(String cliente, double valor) {
        System.out.println("Pagamento: processando R$" + valor + " de " + cliente);
        return true;
    }
}

public class ServicoNotaFiscal {
    public void emitir(String cliente, String produto, double valor) {
        System.out.println("Nota Fiscal: emitida para " + cliente + " — " + produto + " R$" + valor);
    }
}

public class ServicoEmail {
    public void enviarConfirmacao(String cliente, String produto) {
        System.out.println("Email: confirmação enviada para " + cliente + " — pedido: " + produto);
    }
}

public class Main {
    public static void main(String[] args) {
        String cliente = "thiago";
        String produto = "Teclado Mecânico";
        int quantidade = 1;
        double valor = 350.0;

        ServicoEstoque estoque = new ServicoEstoque();
        ServicoPagamento pagamento = new ServicoPagamento();
        ServicoNotaFiscal notaFiscal = new ServicoNotaFiscal();
        ServicoEmail email = new ServicoEmail();

        // o cliente precisa conhecer e orquestrar todos os subsistemas
        if (estoque.verificarDisponibilidade(produto, quantidade)) {
            if (pagamento.processar(cliente, valor)) {
                estoque.reservar(produto, quantidade);
                notaFiscal.emitir(cliente, produto, valor);
                email.enviarConfirmacao(cliente, produto);
            }
        }
    }
}
O Main conhece a ordem exata de cada etapa e instancia todos os subsistemas. Como esconder essa complexidade atrás de uma interface simples, de forma que o cliente precise apenas chamar um único método para processar o pedido?

Crie uma PedidoFacade que encapsule a orquestração dos quatro subsistemas. Após a refatoração, o Main deve se reduzir a uma única linha: facade.realizarPedido(cliente, produto, quantidade, valor).
