Desafio — Case 5

Smell Code para refatorar

Contexto

Uma aplicação corporativa possui um sistema de log que registra eventos em tempo de execução. Cada log passa por duas etapas: primeiro é formatado em um padrão (JSON, Texto simples ou XML), depois é enviado para um destino (Console, Arquivo ou Servidor remoto).

O time de infraestrutura quer adicionar o formato CSV e o destino Banco de dados na próxima versão.

Sobre o smell code

Divergent Change + Violação do SRP

O SistemaDeLog muda por dois motivos completamente diferentes: se um novo formato de log for exigido, a classe muda; se um novo destino de saída for adicionado, a classe também muda.

As duas dimensões são totalmente independentes — qualquer formato deve poder se combinar com qualquer destino — mas estão acopladas dentro do mesmo método.

Código atual

smell code 
public class SistemaDeLog {

    public void registrar(String nivel, String mensagem, String formato, String destino) {

        String logFormatado;

        // Motivo de mudança 1: Formato do log
        if (formato.equals("JSON")) {
            System.out.println("Formatando log como JSON...");
            logFormatado = "{\"nivel\": \"" + nivel + "\", \"mensagem\": \"" + mensagem + "\"}";

        } else if (formato.equals("TEXTO")) {
            System.out.println("Formatando log como texto simples...");
            logFormatado = "[" + nivel + "] " + mensagem;

        } else if (formato.equals("XML")) {
            System.out.println("Formatando log como XML...");
            logFormatado = "<log><nivel>" + nivel + "</nivel><mensagem>" + mensagem + "</mensagem></log>";

        } else {
            throw new IllegalArgumentException("Formato desconhecido.");
        }

        // Motivo de mudança 2: Destino do log
        if (destino.equals("CONSOLE")) {
            System.out.println("Saída no console: " + logFormatado);

        } else if (destino.equals("ARQUIVO")) {
            System.out.println("Escrevendo no arquivo de log...");
            System.out.println("Conteúdo gravado: " + logFormatado);

        } else if (destino.equals("REMOTO")) {
            System.out.println("Enviando log para servidor remoto...");
            System.out.println("Payload enviado: " + logFormatado);

        } else {
            throw new IllegalArgumentException("Destino desconhecido.");
        }
    }
}
Formato e destino variam de forma completamente independente: JSON + CONSOLE, XML + ARQUIVO, TEXTO + REMOTO — qualquer combinação deve ser possível. Como separar essas responsabilidades de forma que adicionar CSV como formato ou Banco de dados como destino não exija tocar em nenhuma classe já existente?

A solução deve usar duas interfaces de estratégia independentes: uma para formatação e outra para destino. O SistemaDeLog deve receber ambas pelo construtor. Atenção: o resultado da estratégia de formatação (String) é a entrada da estratégia de destino — o fluxo entre as duas não é opcional.
