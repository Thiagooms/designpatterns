Desafio — Case 2

Smell Code para refatorar

Contexto

Uma plataforma de BI precisa gerar relatórios para diferentes áreas da empresa. Os dados podem vir de três fontes: API REST, arquivo CSV ou banco de dados. O relatório final pode ser exportado em três formatos: PDF, Excel ou HTML.

O desenvolvedor implementou tudo em uma única classe. O código funciona, mas o produto quer adicionar a fonte GraphQL e o formato de exportação JSON na próxima versão.

Sobre o smell code

Divergent Change + Violação do SRP

O GeradorDeRelatorio muda por dois motivos completamente diferentes: se uma nova fonte de dados for adicionada, a classe muda; se um novo formato de exportação for exigido, a classe também muda. Ela conhece detalhes de infraestrutura (como conectar em cada fonte) e de apresentação (como montar cada formato) ao mesmo tempo.

As duas dimensões de variação são independentes entre si — qualquer fonte deve poder se combinar com qualquer formato — mas estão acopladas dentro do mesmo método.

Código atual

smell code 
import java.util.List;
import java.util.ArrayList;
import java.util.stream.Collectors;

public class GeradorDeRelatorio {

    public void gerarEExportar(String fonte, String formato) {

        List<String> dados = new ArrayList<>();

        // Motivo de mudança 1: Fonte de dados
        if (fonte.equals("API")) {
            System.out.println("Conectando à API REST...");
            dados = List.of("vendas", "estoque", "clientes");

        } else if (fonte.equals("CSV")) {
            System.out.println("Lendo arquivo CSV...");
            dados = List.of("linha1", "linha2", "linha3");

        } else if (fonte.equals("BANCO")) {
            System.out.println("Executando query no banco de dados...");
            dados = List.of("registro1", "registro2", "registro3");

        } else {
            throw new IllegalArgumentException("Fonte desconhecida.");
        }

        // Processamento fixo
        List<String> dadosProcessados = dados.stream()
                .map(String::toUpperCase)
                .toList();

        // Motivo de mudança 2: Formato de exportação
        if (formato.equals("PDF")) {
            System.out.println("Gerando PDF...");
            System.out.println("[PDF] " + dadosProcessados);

        } else if (formato.equals("EXCEL")) {
            System.out.println("Gerando planilha Excel...");
            System.out.println("[EXCEL] " + String.join(";", dadosProcessados));

        } else if (formato.equals("HTML")) {
            System.out.println("Gerando página HTML...");
            System.out.println("<ul>" + dadosProcessados.stream()
                    .map(d -> "<li>" + d + "</li>")
                    .collect(Collectors.joining()) + "</ul>");

        } else {
            throw new IllegalArgumentException("Formato desconhecido.");
        }
    }
}
Fonte e formato variam de forma completamente independente: API + PDF, CSV + HTML, BANCO + Excel — qualquer combinação deve ser possível. Como separar essas duas responsabilidades de forma que adicionar GraphQL como fonte ou JSON como formato não exija tocar em nenhuma classe já existente?

O processamento fixo (toUpperCase) deve continuar no orquestrador. A solução deve usar duas interfaces de estratégia independentes: uma para coleta de dados e outra para exportação. O GeradorDeRelatorio deve receber ambas pelo construtor.
