Desafio — Case 8

Smell Code para refatorar

Contexto

Um banco digital oferece três modalidades de empréstimo: Pessoal, Consignado e Imobiliário. Cada modalidade tem sua própria taxa de juros, forma de calcular o total e processo de registro de contrato.

O desenvolvedor implementou tudo em um único serviço. O código funciona, mas o banco quer adicionar duas novas modalidades na próxima versão: Estudantil e Rural.

Sobre o smell code

Switch Statements + Inappropriate Intimacy

O ServicoEmprestimo instancia diretamente cada modalidade concreta e ainda conhece detalhes internos de cada uma — qual taxa configurar, qual desconto aplicar. Toda nova modalidade exige abrir e modificar essa classe.

Repare que após a criação do objeto, o serviço ainda chama métodos específicos de configuração antes de processar — o que aumenta ainda mais o acoplamento.

Código atual

smell code 
public class ServicoEmprestimo {

    public Emprestimo processar(String modalidade, double valor) {

        Emprestimo emprestimo = null;

        if (modalidade.equalsIgnoreCase("PESSOAL")) {
            emprestimo = new EmprestimoPessoal();
            emprestimo.aplicarTaxaMercado();

        } else if (modalidade.equalsIgnoreCase("CONSIGNADO")) {
            emprestimo = new EmprestimoConsignado();
            emprestimo.aplicarDescontoFolha();

        } else if (modalidade.equalsIgnoreCase("IMOBILIARIO")) {
            emprestimo = new EmprestimoImobiliario();
            emprestimo.aplicarGarantiaImovel();

        } else {
            throw new IllegalArgumentException("Modalidade desconhecida: " + modalidade);
        }

        emprestimo.calcularJuros(valor);
        emprestimo.calcularTotal(valor);
        emprestimo.registrarContrato();

        return emprestimo;
    }
}
Cada modalidade tem sua própria taxa: o Pessoal cobra 3% ao mês, o Consignado cobra 1,5% (desconto em folha) e o Imobiliário cobra 0,7% com garantia do imóvel. Como isolar essas regras sem abrir o ServicoEmprestimo toda vez que uma nova modalidade surgir?

A saída do programa deve continuar sendo a mesma após a refatoração. Leve em conta que a interface deve expor calcularJuros, calcularTotal e registrarContrato como contrato comum entre todas as modalidades.
