Desafio — Case 4

Smell Code para refatorar

Contexto

Uma aplicação web utiliza um GerenciadorDeCache para armazenar dados temporários em memória e evitar consultas repetidas ao banco de dados. Diferentes partes do sistema — serviço de usuários, serviço de produtos e serviço de sessões — precisam ler e escrever nesse cache.

O problema é que cada serviço instancia o seu próprio GerenciadorDeCache, fazendo com que cada um tenha seu próprio estado isolado. Um dado gravado por um serviço nunca é encontrado por outro.

Sobre o smell code

Múltiplas Instâncias de um Recurso Compartilhado

O GerenciadorDeCache é instanciado livremente com new em cada serviço. Como cada instância mantém seu próprio Map interno, o cache fica fragmentado — cada serviço enxerga apenas o que ele mesmo gravou.

O Main abaixo demonstra o problema: o ServicoUsuario grava um dado no cache, mas o ServicoProduto não consegue encontrá-lo porque está usando uma instância diferente.

Código atual

smell code 
import java.util.HashMap;
import java.util.Map;

public class GerenciadorDeCache {

    private Map<String, String> cache = new HashMap<>();

    public void adicionar(String chave, String valor) {
        cache.put(chave, valor);
        System.out.println("Cache gravado: " + chave + " = " + valor);
    }

    public String buscar(String chave) {
        return cache.getOrDefault(chave, "NÃO ENCONTRADO");
    }

    public int tamanho() {
        return cache.size();
    }
}

public class ServicoUsuario {
    private GerenciadorDeCache cache = new GerenciadorDeCache(); // instância própria

    public void salvarSessao(String usuario) {
        cache.adicionar("sessao_" + usuario, "ativa");
    }
}

public class ServicoProduto {
    private GerenciadorDeCache cache = new GerenciadorDeCache(); // instância própria

    public void verificarSessao(String usuario) {
        String sessao = cache.buscar("sessao_" + usuario);
        System.out.println("Sessão de " + usuario + ": " + sessao);
    }
}

public class Main {
    public static void main(String[] args) {
        ServicoUsuario usuario = new ServicoUsuario();
        ServicoProduto produto = new ServicoProduto();

        usuario.salvarSessao("thiago");
        produto.verificarSessao("thiago");

        // Saída: "Sessão de thiago: NÃO ENCONTRADO"
        // O dado foi gravado em uma instância, buscado em outra.
    }
}
O cache está fragmentado em múltiplas instâncias. Como garantir que todos os serviços compartilhem exatamente o mesmo GerenciadorDeCache, sem que nenhum possa criar uma nova instância por engano?

Refatore o GerenciadorDeCache para Singleton com double-checked locking. Após a refatoração, a saída do Main deve ser "Sessão de thiago: ativa" — provando que todos os serviços compartilham a mesma instância.
