Desafio — Case 5

Smell Code para refatorar

Contexto

Uma API possui um ContadorDeRequisicoes responsável por registrar quantas requisições cada rota recebeu. Dois controllers — o de usuários e o de pedidos — precisam incrementar e consultar esse contador.

O problema é que cada controller instancia o seu próprio ContadorDeRequisicoes. Com isso, o total de requisições nunca é acumulado corretamente — cada controller enxerga apenas o que ele mesmo registrou.

Sobre o smell code

Múltiplas Instâncias de um Recurso Compartilhado

O ContadorDeRequisicoes é instanciado com new dentro de cada controller. Como cada instância mantém seu próprio Map interno de contagens, os dados ficam isolados por controller.

O Main abaixo demonstra o problema: após o ControllerUsuario e o ControllerPedido registrarem requisições, o total geral é consultado em uma terceira instância — e retorna 0, pois nenhuma das contagens foi compartilhada.

Código atual

smell code 
import java.util.HashMap;
import java.util.Map;

public class ContadorDeRequisicoes {

    private Map<String, Integer> contagens = new HashMap<>();

    public void registrar(String rota) {
        contagens.merge(rota, 1, Integer::sum);
        System.out.println("Requisição registrada: " + rota);
    }

    public int total() {
        return contagens.values().stream().mapToInt(Integer::intValue).sum();
    }

    public void relatorio() {
        contagens.forEach((rota, qtd) ->
            System.out.println(rota + ": " + qtd + " requisições")
        );
    }
}

public class ControllerUsuario {
    private ContadorDeRequisicoes contador = new ContadorDeRequisicoes(); // instância própria

    public void listarUsuarios() {
        contador.registrar("/usuarios");
    }
}

public class ControllerPedido {
    private ContadorDeRequisicoes contador = new ContadorDeRequisicoes(); // instância própria

    public void listarPedidos() {
        contador.registrar("/pedidos");
    }
}

public class Main {
    public static void main(String[] args) {
        ControllerUsuario usuarios = new ControllerUsuario();
        ControllerPedido pedidos = new ControllerPedido();

        usuarios.listarUsuarios();
        usuarios.listarUsuarios();
        pedidos.listarPedidos();

        ContadorDeRequisicoes contador = new ContadorDeRequisicoes(); // terceira instância!
        System.out.println("Total de requisições: " + contador.total()); // imprime 0
        contador.relatorio(); // não imprime nada
    }
}
O contador está fragmentado em múltiplas instâncias. Como garantir que todos os controllers e o Main compartilhem exatamente o mesmo ContadorDeRequisicoes, acumulando as contagens corretamente?

Refatore o ContadorDeRequisicoes para Singleton com double-checked locking. Após a refatoração, a saída do Main deve ser "Total de requisições: 3" e o relatório deve exibir /usuarios: 2 requisições e /pedidos: 1 requisições.
