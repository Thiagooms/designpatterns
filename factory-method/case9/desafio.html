Desafio — Case 9

Smell Code para refatorar

Contexto

Uma plataforma de e-commerce precisa enviar notificações para os clientes em três canais: E-mail, SMS e Push. Cada canal possui sua própria forma de configurar credenciais, formatar a mensagem e registrar o envio.

O desenvolvedor implementou tudo em um único serviço. O código funciona, mas o produto quer adicionar dois novos canais na próxima versão: WhatsApp e Webhook.

Sobre o smell code

Switch Statements + Inappropriate Intimacy

O ServicoNotificacao instancia diretamente cada canal concreto e ainda conhece detalhes internos de cada um — qual provedor configurar, qual template aplicar. Toda vez que um novo canal é adicionado, essa classe precisa ser aberta e modificada.

Repare que após a criação do objeto, o serviço ainda chama métodos de configuração específicos de cada canal antes de enviar — o que aumenta ainda mais o acoplamento.

Código atual

smell code 
public class ServicoNotificacao {

    public Notificacao enviar(String canal, String destinatario, String mensagem) {

        Notificacao notificacao = null;

        if (canal.equalsIgnoreCase("EMAIL")) {
            notificacao = new NotificacaoEmail();
            notificacao.configurarSMTP();

        } else if (canal.equalsIgnoreCase("SMS")) {
            notificacao = new NotificacaoSMS();
            notificacao.configurarProvedor();

        } else if (canal.equalsIgnoreCase("PUSH")) {
            notificacao = new NotificacaoPush();
            notificacao.configurarDispositivo();

        } else {
            throw new IllegalArgumentException("Canal desconhecido: " + canal);
        }

        notificacao.formatarMensagem(destinatario, mensagem);
        notificacao.enviar();
        notificacao.registrarEnvio();

        return notificacao;
    }
}
Cada canal tem suas particularidades: o E-mail exige configuração de servidor SMTP, o SMS depende de um provedor externo com chave de API e o Push precisa do token do dispositivo. Como isolar essas responsabilidades sem abrir o ServicoNotificacao toda vez que um novo canal surgir?

A saída do programa deve continuar sendo a mesma após a refatoração. Leve em conta que a interface deve expor formatarMensagem, enviar e registrarEnvio como contrato comum entre todos os canais.
