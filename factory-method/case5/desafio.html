Desafio — Case 5

Smell Code para refatorar

Contexto

Um sistema de e-commerce precisa processar pagamentos. Atualmente suporta três métodos: Cartão de Crédito, Boleto e PIX.

O desenvolvedor implementou tudo dentro de um único serviço. O código funciona, mas o time de produto já solicitou a integração de dois novos métodos de pagamento para o próximo trimestre: Cartão de Débito e Criptomoeda.

Sobre o smell code

Switch Statements + Inappropriate Intimacy

O ServicoPagamento conhece e instancia diretamente todas as implementações concretas de pagamento. Cada novo método de pagamento obriga o desenvolvedor a abrir essa classe e modificá-la — violando o princípio aberto para extensão, fechado para modificação.

Repare nos new dentro de cada bloco do if/else if: a responsabilidade de criação dos objetos está misturada com a lógica de negócio.

Código atual

smell code 
public class ServicoPagamento {

    public double processar(String metodo, double valor) {

        if (metodo.equals("cartao")) {
            PagamentoCartao p = new PagamentoCartao();
            return p.calcularTotal(valor);

        } else if (metodo.equals("boleto")) {
            PagamentoBoleto p = new PagamentoBoleto();
            return p.calcularTotal(valor);

        } else if (metodo.equals("pix")) {
            PagamentoPix p = new PagamentoPix();
            return p.calcularTotal(valor);

        } else {
            throw new IllegalArgumentException("Método desconhecido: " + metodo);
        }
    }
}
public class Main {

    public static void main(String[] args) {
        ServicoPagamento servico = new ServicoPagamento();

        System.out.println(servico.processar("cartao", 100.0));
        System.out.println(servico.processar("boleto", 100.0));
        System.out.println(servico.processar("pix",    100.0));
    }
}
Cada método de pagamento aplica uma regra diferente sobre o valor: o Cartão de Crédito cobra uma taxa percentual de 3,5% sobre o valor, o Boleto cobra uma taxa fixa de R$ 3,50 independente do valor, e o PIX não cobra nenhuma taxa — o valor cobrado é o valor original. Como você vai isolar essas regras sem encostar no ServicoPagamento toda vez?

A saída do programa deve continuar sendo a mesma após a refatoração. O método retorna double — leve isso em conta ao definir o contrato da interface.
