Desafio — Case 6

Smell Code para refatorar

Contexto

Uma aplicação web precisa autenticar usuários. Atualmente suporta três estratégias: JWT, OAuth e Session.

O desenvolvedor implementou tudo dentro de um único serviço. O código funciona, mas o time de segurança solicitou a adição de autenticação por API Key para integrações B2B na próxima sprint.

Sobre o smell code

Switch Statements + Inappropriate Intimacy

O ServicoAutenticacao conhece e instancia diretamente todas as estratégias concretas de autenticação. Cada nova estratégia obriga o desenvolvedor a abrir essa classe e modificá-la — violando o princípio aberto para extensão, fechado para modificação.

Repare nos new dentro de cada bloco do if/else if: a responsabilidade de criação dos objetos está misturada com a lógica de negócio. O serviço não deveria saber qual estratégia concreta instanciar — apenas como usá-la.

Código atual

smell code 
public class ServicoAutenticacao {

    public String autenticar(String estrategia, String usuario, String credencial) {

        if (estrategia.equals("jwt")) {
            AutenticacaoJWT a = new AutenticacaoJWT();
            return a.autenticar(usuario, credencial);

        } else if (estrategia.equals("oauth")) {
            AutenticacaoOAuth a = new AutenticacaoOAuth();
            return a.autenticar(usuario, credencial);

        } else if (estrategia.equals("session")) {
            AutenticacaoSession a = new AutenticacaoSession();
            return a.autenticar(usuario, credencial);

        } else {
            throw new IllegalArgumentException("Estratégia desconhecida: " + estrategia);
        }
    }
}
public class Main {

    public static void main(String[] args) {
        ServicoAutenticacao servico = new ServicoAutenticacao();

        System.out.println(servico.autenticar("jwt",     "thiago", "senha123"));
        System.out.println(servico.autenticar("oauth",   "thiago", "token-google-xyz"));
        System.out.println(servico.autenticar("session", "thiago", "senha123"));
    }
}
Cada estratégia retorna um token diferente: o JWT gera um token assinado com os dados do usuário, o OAuth troca o token externo por um token interno da aplicação, e o Session cria uma sessão no servidor e retorna o ID da sessão. Como você vai isolar essas lógicas sem encostar no ServicoAutenticacao toda vez que uma nova estratégia surgir?

A saída do programa deve continuar sendo a mesma após a refatoração. O método retorna String — leve isso em conta ao definir o contrato da interface.
