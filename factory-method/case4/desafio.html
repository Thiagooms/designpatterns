Desafio — Case 4

Smell Code para refatorar

Contexto

Um sistema de e-commerce precisa calcular o frete dos pedidos. Atualmente suporta três transportadoras: Correios, FedEx e Transportadora Local.

O desenvolvedor implementou tudo dentro de um único serviço. O código funciona, mas a equipe já recebeu demanda para integrar mais duas transportadoras no próximo mês.

Sobre o smell code

Switch Statements + Inappropriate Intimacy

O mesmo smell do case anterior, mas agora aplicado a um contexto diferente para fixar o reconhecimento. Repare nos new dentro de cada bloco do if/else if: o serviço conhece e instancia diretamente todas as transportadoras concretas, acoplando a lógica de criação à lógica de negócio.

Cada vez que uma nova transportadora precisar ser adicionada ou uma existente precisar mudar sua forma de calcular o frete, essa classe precisará ser aberta e modificada — violando o princípio de que uma classe deve estar aberta para extensão, fechada para modificação.

Código atual

smell code 
public class ServicoFrete {

    public double calcular(String transportadora, double peso, String destino) {

        if (transportadora.equals("correios")) {
            FreteCorreios f = new FreteCorreios();
            return f.calcularFrete(peso, destino);

        } else if (transportadora.equals("fedex")) {
            FreteFedEx f = new FreteFedEx();
            return f.calcularFrete(peso, destino);

        } else if (transportadora.equals("local")) {
            FreteLocal f = new FreteLocal();
            return f.calcularFrete(peso, destino);

        } else {
            throw new IllegalArgumentException("Transportadora desconhecida: " + transportadora);
        }
    }
}
public class Main {

    public static void main(String[] args) {
        ServicoFrete servico = new ServicoFrete();

        System.out.println(servico.calcular("correios", 2.5, "SP"));
        System.out.println(servico.calcular("fedex",   2.5, "SP"));
        System.out.println(servico.calcular("local",   2.5, "SP"));
    }
}
Cada transportadora tem sua própria fórmula de cálculo. Correios cobra por peso e distância, FedEx aplica uma taxa fixa internacional, e a Local tem desconto para entregas na mesma cidade. Como você vai isolar essas regras sem encostar no ServicoFrete toda vez?

A saída do programa deve continuar sendo a mesma após a refatoração. O método de cálculo retorna double — leve isso em conta ao definir o contrato da interface.
