Desafio — Case 3

Smell Code para refatorar

Contexto

Um sistema de infraestrutura precisa gerenciar arquivos gerados pela aplicação. Antes de armazenar, cada arquivo passa por um algoritmo de compressão: ZIP, GZIP ou nenhum. O arquivo comprimido é então enviado para um destino de armazenamento: disco local, bucket S3 ou servidor FTP.

O time de infraestrutura quer adicionar suporte ao algoritmo BZIP2 e ao destino Google Cloud Storage na próxima sprint.

Sobre o smell code

Divergent Change + Violação do SRP

O GerenciadorDeArquivos muda por dois motivos completamente diferentes: se um novo algoritmo de compressão for adicionado, a classe muda; se um novo destino de armazenamento for suportado, a classe também muda.

As duas dimensões são totalmente independentes — qualquer algoritmo deve poder se combinar com qualquer destino — mas estão acopladas dentro do mesmo método.

Código atual

smell code 
public class GerenciadorDeArquivos {

    public void comprimirEArmazenar(String conteudo, String algoritmo, String destino) {

        String conteudoComprimido;

        // Motivo de mudança 1: Algoritmo de compressão
        if (algoritmo.equals("ZIP")) {
            System.out.println("Comprimindo com ZIP...");
            conteudoComprimido = "[ZIP]" + conteudo;

        } else if (algoritmo.equals("GZIP")) {
            System.out.println("Comprimindo com GZIP...");
            conteudoComprimido = "[GZIP]" + conteudo;

        } else if (algoritmo.equals("NENHUM")) {
            System.out.println("Sem compressão, mantendo arquivo original...");
            conteudoComprimido = conteudo;

        } else {
            throw new IllegalArgumentException("Algoritmo desconhecido.");
        }

        // Motivo de mudança 2: Destino de armazenamento
        if (destino.equals("LOCAL")) {
            System.out.println("Salvando arquivo no disco local...");
            System.out.println("Conteúdo gravado: " + conteudoComprimido);

        } else if (destino.equals("S3")) {
            System.out.println("Conectando ao bucket S3...");
            System.out.println("Upload concluído: " + conteudoComprimido);

        } else if (destino.equals("FTP")) {
            System.out.println("Abrindo conexão FTP na porta 21...");
            System.out.println("Upload concluído: " + conteudoComprimido);

        } else {
            throw new IllegalArgumentException("Destino desconhecido.");
        }
    }
}
Algoritmo e destino variam de forma completamente independente: ZIP + S3, GZIP + FTP, NENHUM + LOCAL — qualquer combinação deve ser possível. Como separar essas responsabilidades de forma que adicionar BZIP2 ou Google Cloud Storage não exija tocar em nenhuma classe já existente?

A solução deve usar duas interfaces de estratégia independentes: uma para compressão e outra para armazenamento. O GerenciadorDeArquivos deve receber ambas pelo construtor e apenas orquestrar — sem nenhum if/else.
